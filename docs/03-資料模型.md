# 03 · 資料模型（Supabase / Postgres）

> 改動資料結構前先讀本文件，並在改完後同步更新此處。所有表都要有 RLS。
>
> **可執行版本**：完整、可貼進 Supabase SQL Editor 直接跑（且可重跑）的腳本在 `supabase/schema.sql`，本文件為說明性的真實來源；兩者需保持一致。
>
> **配對採「旅程層級邀請」**（已與使用者確認）：不做帳號層級配對表；每趟 `trips.invite_code`，夥伴用碼加入該趟 `trip_members`。建立/加入一律走下方 `create_trip` / `join_trip_by_code` RPC。

## ER 關係概覽

```
auth.users (Supabase 內建)
   │
   └─< profiles            個人資料（display_name, avatar_url）

trips                       一趟旅行
   ├─< trip_members         趟次成員（兩人共編的關鍵）
   ├─< days                 行程的每一天
   ├─< items                行程項目（定點/區域/書籤）
   │     └─< area_candidates  區域內的候選店家（不排序）
   ├─< transports           相鄰項目間的交通
   ├─< documents            文件匣（中繼資料，檔案在 Storage）
   │     ├─< document_items       文件 ↔ 行程項目（多對多）
   │     └─< document_transports  文件 ↔ 交通（多對多）
   ├─< packing_items        行李清單（依個人分）
   └─< activity_log         改動紀錄（給通知用）
```

## 核心列舉值

- `items.type`：`point` | `area`
- `items.status`：`bookmark`（地圖書籤，未排入）｜`scheduled`（已排入某天）
- `transports.mode`：`walk` | `transit` | `drive` | `bike` | `custom`
- `documents.category`：`flight`（機票）｜`lodging`（住宿）｜`document`（文件/簽證保險）｜`other`（其他）
- `trip_members.role`：`owner` | `member`

## 建表 SQL

```sql
-- 個人資料（對應 auth.users）
create table profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  display_name text,
  avatar_url text,
  created_at timestamptz default now()
);

-- 旅程
create table trips (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  destination text,
  start_date date,
  end_date date,
  cover_image_url text,
  invite_code text unique,              -- 旅程層級邀請碼（夥伴輸入此碼加入；由 create_trip 產生）
  created_by uuid references auth.users(id),
  created_at timestamptz default now()
);

-- 趟次成員（兩人共編）
create table trip_members (
  trip_id uuid references trips(id) on delete cascade,
  user_id uuid references auth.users(id) on delete cascade,
  role text not null default 'member',  -- owner | member
  joined_at timestamptz default now(),
  primary key (trip_id, user_id)
);

-- 每一天
create table days (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid references trips(id) on delete cascade,
  date date not null,
  day_index int not null,               -- Day1=1, Day2=2...
  created_at timestamptz default now(),
  unique (trip_id, day_index)           -- 同趟同一天唯一，防併發/StrictMode 重複建立
);

-- 行程項目（定點 / 區域 / 書籤）
create table items (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid references trips(id) on delete cascade,
  day_id uuid references days(id) on delete set null,  -- null = 尚未排入（書籤）
  type text not null,                   -- point | area
  status text not null default 'scheduled', -- bookmark | scheduled
  name text not null,
  lat double precision,
  lng double precision,
  google_place_id text,
  photo_url text,
  scheduled_time time,                  -- 可選填時間（定點常用）
  time_slot text,                       -- 區域的時段，如 '下午'
  radius_m int,                         -- 區域圓形半徑（type=area 時）
  category text,                        -- 自由標記，如 '逛街' '夜市'
  order_index int not null default 0,   -- 同一天內排序
  notes text,
  created_by uuid references auth.users(id),
  created_at timestamptz default now()
);

-- 區域內的候選店家（不排序）
create table area_candidates (
  id uuid primary key default gen_random_uuid(),
  item_id uuid references items(id) on delete cascade,  -- 必為 type=area 的 item
  name text not null,
  lat double precision,
  lng double precision,
  google_place_id text,
  chosen boolean default false,         -- 當天決定去這間
  notes text,
  created_at timestamptz default now()
);

-- 相鄰項目間的交通
create table transports (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid references trips(id) on delete cascade,
  from_item_id uuid references items(id) on delete cascade,
  to_item_id uuid references items(id) on delete cascade,
  mode text not null,                   -- walk | transit | drive | bike | custom
  duration_min int,
  distance_m int,
  custom_label text,                    -- 自定義時用：'新幹線' '包車'
  cost_text text,                       -- 費用顯示字串，如 '¥240'（階段 3；幣別多樣故存純顯示字串）
  route_polyline text,                  -- Google 路線編碼（可選快取，畫真實路線用）
  notes text,
  created_at timestamptz default now(),
  unique (from_item_id, to_item_id)     -- 一段交通一列（以相鄰對為鍵 upsert，防併發重複）
);

-- 文件匣（中繼資料；實際檔案在 Storage）。連結改走多對多關聯表（見下方），故無 linked_item_id。
create table documents (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid references trips(id) on delete cascade,
  category text not null,               -- flight | lodging | document | other
  file_name text not null,
  storage_path text not null,           -- Storage 內路徑（透過 storage 抽象層存取）
  uploaded_by uuid references auth.users(id),
  created_at timestamptz default now()
);

-- 文件 ↔ 行程項目 / 交通 的多對多連結（在詳情頁用 toggle 管理增刪）
create table document_items (
  document_id uuid references documents(id) on delete cascade,
  item_id uuid references items(id) on delete cascade,
  created_at timestamptz default now(),
  primary key (document_id, item_id)
);
create table document_transports (
  document_id uuid references documents(id) on delete cascade,
  transport_id uuid references transports(id) on delete cascade,
  created_at timestamptz default now(),
  primary key (document_id, transport_id)
);

-- 行李清單（依個人分）
create table packing_items (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid references trips(id) on delete cascade,
  owner_user_id uuid references auth.users(id) on delete cascade,  -- 誰的行李
  name text not null,
  category text,                        -- 證件 / 電子產品 / 盥洗...
  is_packed boolean default false,
  created_at timestamptz default now()
);

-- 改動紀錄（給「對方改動通知」用）
create table activity_log (
  id uuid primary key default gen_random_uuid(),
  trip_id uuid references trips(id) on delete cascade,
  user_id uuid references auth.users(id),   -- 誰做的
  action text not null,                 -- 如 'item.added' 'item.reordered'
  target_summary text,                  -- 人話摘要，如 '新增了 心齋橋'
  created_at timestamptz default now()
);
```

## 函式與 RPC

> 皆 `security definer` 並 `set search_path = public`。建立/加入旅程走 RPC，以保證原子性、伺服器端產生唯一邀請碼、並強制兩人上限。完整內文見 `supabase/schema.sql`。

- `is_trip_member(p_trip_id)` → bool：目前使用者是否為該趟成員（policy 用；definer 故查 `trip_members` 不觸發自身 RLS、不遞迴）。
- `is_co_member(p_user)` → bool：目前使用者是否與某人同趟（給 `profiles` 互看用）。
- `handle_new_user()` + `on auth.users` after-insert 觸發器：新帳號（Email/Google）自動建 `profiles`。
- `create_trip(p_name, p_destination, p_start_date, p_end_date)` → trips：產生唯一 6 碼 `invite_code` + 把自己加為 `trip_members(owner)`，原子完成並回傳該趟。
- `join_trip_by_code(p_code)` → trips：驗證邀請碼、防重複加入、限成員 < 2，插入 `trip_members(member)` 並回傳該趟。

## Row Level Security（RLS）原則

所有「屬於某趟旅程」的表，存取條件都是「**目前使用者是該 trip 的成員**」。下為原則與關鍵差異點；**所有表的完整 policy 見 `supabase/schema.sql`**（profiles、trip_members、days、items、area_candidates、transports、documents、packing_items、activity_log 全數 enable 並建 policy）。

```sql
alter table trips enable row level security;
alter table items enable row level security;
-- ...其餘表同樣 enable

-- 輔助：判斷目前使用者是否為某 trip 的成員
create or replace function is_trip_member(p_trip_id uuid)
returns boolean language sql security definer stable as $$
  select exists (
    select 1 from trip_members
    where trip_id = p_trip_id and user_id = auth.uid()
  );
$$;

-- trips：成員可讀；建立者可改/刪。
-- 注意：不開放 client 直接 insert——建立一律走 create_trip RPC，
-- 否則可能產生「無邀請碼、無成員」的孤兒旅程。
create policy "members read trips" on trips
  for select using (is_trip_member(id));
create policy "creator update trips" on trips
  for update using (created_by = auth.uid()) with check (created_by = auth.uid());
create policy "creator delete trips" on trips
  for delete using (created_by = auth.uid());

-- trip_members：成員可讀名單、可移除自己；加入成員一律走 RPC（不開放 client insert）
create policy "members read trip_members" on trip_members
  for select using (is_trip_member(trip_id));
create policy "leave own membership" on trip_members
  for delete using (user_id = auth.uid());

-- profiles：本人或同趟夥伴可讀；本人可寫（夥伴資料唯讀）
create policy "read self or co-member profiles" on profiles
  for select using (id = auth.uid() or is_co_member(id));
create policy "update own profile" on profiles
  for update using (id = auth.uid()) with check (id = auth.uid());

-- items（其餘 trip 子表比照：用 is_trip_member(trip_id)）
create policy "members rw items" on items
  for all using (is_trip_member(trip_id)) with check (is_trip_member(trip_id));

-- packing_items：成員都看得到（互看），但只能改自己的
create policy "members read packing" on packing_items
  for select using (is_trip_member(trip_id));
create policy "owner write packing" on packing_items
  for all using (owner_user_id = auth.uid()) with check (owner_user_id = auth.uid());
```

> 注意：`area_candidates` 沒有 trip_id 欄位，RLS 需透過其 `item_id` join 回 items 判斷成員。

## 設計備註

- **每日（days）唯一鍵**：`(trip_id, day_index)` 唯一。前端 `ensureDays` 是「先查無、再建立」非原子操作，React StrictMode（dev 重複觸發 effect）或兩人同時首開同趟時會併發 insert；唯一鍵讓第二次 insert 撞 `23505` 後被前端略過、改讀現有，避免重複日。`schema.sql` 內含自我修復段（清重複日＋補唯一鍵，可重跑）。
- **書籤** = `items` 裡 `status='bookmark'` 且 `day_id is null`；地圖依 status 上色。
- **區域候選店家**不排序，故 `area_candidates` 無 order 欄位。
- **交通**綁在「from→to 兩個 item」上，以 `(from_item_id, to_item_id)` 唯一鍵 **upsert**（一段一列、防兩人併發重複）。前端依「目前相鄰對」查表顯示：**重排不破壞 DB** —— 不再相鄰的 transport 查不到、UI 顯示為「加交通」，但保留在 DB（相鄰關係復原即自動再現，等於記住該段）；item 刪除時 FK `on delete cascade` 自動清掉相關 transport。`cost_text` 為費用顯示字串（大眾運輸 Directions 有 `fare` 時自動帶入）；`route_polyline` 快取 Directions 路線供主地圖畫實線。
- **即時同步與改動通知（階段 6）**：採 Supabase Realtime（postgres_changes）。publication `supabase_realtime` 含六表：`items`、`area_candidates`、`transports`、`documents`、`packing_items`、`activity_log`（見 schema.sql 第 7 節，冪等可重跑）。前端在 `TripRealtimeProvider`（`src/lib/tripRealtime.tsx`）建單一 channel：
  - **鐵律：事件 payload 一律不讀內容、只當「該表變了」的訊號 → 各分頁依 tripId refetch**。原因：INSERT/UPDATE 事件逐訂閱者套 RLS，但 **DELETE 事件不套 RLS、不吃 filter、old record 只有主鍵**（官方行為），payload 不可信也不完整；refetch 本身受 RLS 保護。也因此不需 `replica identity full`。
  - `area_candidates` 無 `trip_id` 欄位 → 訂閱不掛 filter，INSERT/UPDATE 靠 RLS 限縮到自己的趟次。
  - **activity_log**：mutation 成功後由前端寫入（`src/lib/activity.ts`，fire-and-forget 不阻斷主流程）。`action` 代碼：`item_add / item_move / item_remove / candidate_add / transport_set / transport_remove / doc_add / doc_remove`；`target_summary` 存**不含人名的完整動作句**（如 `把「梅田藍天大廈」加到 Day 2`），UI 渲染時前綴操作者名字。**行李清單不寫 activity_log**（勾選高頻會洗版），僅靠 Realtime 同步進度。
  - **通知 UI**：對方的 activity_log INSERT → 深色 banner（4 秒自動消失）＋鈴鐺紅點；點鈴鐺開「最近改動」清單並標記已讀。未讀判斷：`localStorage['cotravel:activitySeen:<tripId>']` 存最後已讀的 `created_at`，與對方最新一筆比較（存伺服器時間而非 client now，避免兩端時鐘誤差）。
  - 衝突策略為 **last-write-wins**：不做欄位級 merge，收到事件即 refetch 覆蓋本地。
- **兩人配對（旅程層級）**：`create_trip` 建立 trip 時把自己加進 `trip_members(role=owner)` 並產生 `invite_code`；夥伴用該碼呼叫 `join_trip_by_code` 加入為 `member`（限兩人）。不做帳號層級的常駐配對關係。
- **文件匣 Storage（階段 4）**：`documents` 只存中繼資料，實際檔案放在 private 的 Storage bucket **`documents`**。檔案一律透過 `src/lib/storage/` 抽象層存取（不在元件直接呼叫 Supabase Storage，便於日後換 R2）。路徑慣例 **`<trip_id>/<uuid>.<副檔名>`**：首段即 `trip_id`，`storage.objects` 的 RLS 以 `is_trip_member(((storage.foldername(name))[1])::uuid)` 判斷成員（讀/寫/刪同條件，見 `supabase/schema.sql` 第 6 段）。key 不放原始檔名（Supabase Storage key 不接受中文等字元，會報 Invalid key），原始檔名存 `documents.file_name` 供顯示。離線快取走**應用層 IndexedDB**（`src/lib/offline/docCache.ts`），每份文件由使用者手動下載，與 Storage / Service Worker 無關。
- **文件連結（多對多）**：文件與行程項目、交通的連結走關聯表 `document_items`、`document_transports`（一項目/交通可連多文件、一文件可連多項目/交通）。在行程／交通**詳情頁**用 toggle 管理增刪；關聯表 RLS 透過 join 回 `documents` / `items` / `transports` 判斷成員（見 schema.sql）。項目/交通/文件刪除時，關聯列以 FK `on delete cascade` 自動清除。
