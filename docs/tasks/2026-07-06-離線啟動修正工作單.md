## 1. 症狀

App 是 PWA，文件匣已做「手動下載後離線可看」（IndexedDB 存 Blob），地圖分頁也有離線行程快照。但**實際斷網時整個 App 連首頁都進不去**——已下載的文件明明在本機，卻因為前面的頁面（登入判斷、旅程列表）離線壞掉而永遠打不開。

## 2. 成因診斷（已逐檔查證，2026-07-06）

離線開啟 App 時的啟動鏈，每一環的行為：

| 環節 | 檔案 | 離線行為 | 結果 |
|---|---|---|---|
| ① App shell（HTML/JS/CSS） | `vite.config.ts:90-97` | workbox 已預快取（globPatterns 含 html/js/css） | 應可載入（任務 0 驗證） |
| ② 登入狀態判斷 | `src/lib/auth.tsx:25-40` | `supabase.auth.getSession()` 在 access token 已過期時會嘗試**網路刷新**：可能長時間不 resolve（畫面卡「載入中…」），或 resolve 成 null | **卡住或被當成未登入** |
| ③ 路由守衛 | `src/lib/auth.tsx:92-102` | session 為 null → 導向 `/login` | 被踢到登入頁 |
| ④ 登入頁 | `src/routes/Login.tsx:35` | Google OAuth 與 Email 登入都需要網路 | **死路** |
| ⑤ 旅程列表 | `src/routes/TripList.tsx:43-70` | `listMyTrips()` 網路失敗 → error 畫面只有「重新載入」鈕 | **死路** |
| ⑥ 旅程主畫面 | `src/routes/TripMain.tsx` | 本身不抓資料，可渲染 | 正常 |
| ⑦ 地圖分頁 | `src/routes/tabs/MapTab.tsx:809-819` | 已有 OfflineItinerary 離線快照路徑 | 正常（前提是走得到） |
| ⑧ 文件分頁 | `src/routes/tabs/DocsTab.tsx:68-98` | 初載 `getTripWithMembers`+`listDocuments` 走網路，失敗 → error 死路 | **已下載的檔案打不開** |
| ⑨ 字體 | `index.html:14-19` | Google Fonts CDN 載不到，退回系統字型 | 可接受（任務 4 可選修） |

**結論**：問題不在 service worker，而在 ②③④⑤⑧ 五個資料環節沒有離線後備。修法＝「上次成功載入的資料留一份本地快照，離線時以唯讀模式呈現」。

## 3. 修正原則（必守）

1. **Surgical**：只改指定檔案，不順手重構、不改樣式、不動 `supabase/`、不動既有 IndexedDB 的 `VERSION`。
2. **線上行為零改變**：所有離線後備只在「網路請求失敗」或 `!navigator.onLine` 時啟動。特別注意 `auth.tsx:28` 的 `getSession()` 同時負責 **OAuth 回跳的 PKCE code 交換**——OAuth 回跳必然在線上，所以「線上路徑」的邏輯一個字都不能動，離線超時邏輯必須用 `!navigator.onLine` 守住。
3. **Fallback 以「請求失敗」為準**，`navigator.onLine` 只拿來做 UI 提示與 auth 超時開關（onLine 為 true 不代表真的有網路）。
4. 離線模式一律**唯讀**：所有寫入入口（建立、加入、上傳、選單）隱藏或 disabled。
5. 新檔案的註解風格照抄 `src/lib/offline/itineraryCache.ts`。

## 任務 1：離線登入狀態（②③④）

### 1a. 新檔 `src/lib/offline/lastAuth.ts`

```ts
// 最後一次成功登入的使用者摘要（localStorage）。
// 離線開 App 時 getSession() 可能因 token 過期刷新卡住或回 null，
// 以此判定「本機曾登入過」→ 進入唯讀離線模式，而不是把使用者踢去無法登入的登入頁。
export interface LastAuth {
  id: string
  name: string
  avatarUrl: string | null
}

const KEY = 'cotravel.lastAuth'

export function saveLastAuth(a: LastAuth): void {
  try {
    localStorage.setItem(KEY, JSON.stringify(a))
  } catch {
    // 寫入失敗（容量/隱私模式）不致命，只是下次離線進不了離線模式
  }
}

export function loadLastAuth(): LastAuth | null {
  try {
    const raw = localStorage.getItem(KEY)
    return raw ? (JSON.parse(raw) as LastAuth) : null
  } catch {
    return null
  }
}

export function clearLastAuth(): void {
  localStorage.removeItem(KEY)
}
```

### 1b. 改 `src/lib/auth.tsx`

1. `AuthContextValue` 介面加一個欄位：`offlineUser: LastAuth | null`（import 自 1a）。
2. `AuthProvider` 加 state：`const [offlineUser, setOfflineUser] = useState<LastAuth | null>(null)`。
3. 把初始載入的 useEffect（現在的 25-40 行）改成：**線上路徑照舊**；離線時给 `getSession()` 2.5 秒超時，超時或無 session 就改用 `loadLastAuth()`：

```ts
useEffect(() => {
  let active = true
  const finish = (s: Session | null) => {
    if (!active) return
    setSession(s)
    setLoading(false)
  }
  // 初次載入：讀現有 session（OAuth 回跳的 code 也會在此被 supabase-js 交換）
  const sessionPromise = supabase.auth.getSession().then(({ data }) => data.session)
  if (!navigator.onLine) {
    // 離線：token 過期時 getSession 會等網路刷新而卡住——2.5 秒沒回就走離線授權，
    // 之後若 promise 仍回來（含恢復連線後），下面的 then 會補上真 session
    const timeout = new Promise<'timeout'>((r) => setTimeout(() => r('timeout'), 2500))
    void Promise.race([sessionPromise, timeout]).then((res) => {
      if (!active) return
      if (res === 'timeout' || res === null) {
        setOfflineUser(loadLastAuth())
        setLoading(false)
      } else {
        finish(res)
      }
    })
    sessionPromise.then((s) => {
      if (active && s) {
        setOfflineUser(null)
        setSession(s)
      }
    }).catch(() => {})
  } else {
    sessionPromise.then(finish).catch(() => finish(null))
  }
  const { data: sub } = supabase.auth.onAuthStateChange((_event, next) => {
    setSession(next)
    if (next) setOfflineUser(null)
  })
  return () => {
    active = false
    sub.subscription.unsubscribe()
  }
}, [])
```

4. 在既有的「登入後確保 profile 存在」useEffect（45-53 行）開頭加寫入 lastAuth（欄位取法照抄 `TripList.tsx:82-85` 的 metadata 邏輯）：

```ts
const meta = (user.user_metadata ?? {}) as Record<string, unknown>
saveLastAuth({
  id: user.id,
  name: (meta.full_name as string) || (meta.name as string) || user.email?.split('@')[0] || '我',
  avatarUrl: (meta.avatar_url as string) || (meta.picture as string) || null,
})
```

5. `signOut()` 成功後呼叫 `clearLastAuth()` 並 `setOfflineUser(null)`。
6. context value 加 `offlineUser`。
7. `RequireAuth`：`const { session, loading, offlineUser } = useAuth()`，放行條件改為 `if (!session && !offlineUser) return <Navigate .../>`。

### 1c. 改 `src/routes/Login.tsx:35`

已登入跳轉條件加上離線授權：


## 任務 2：旅程列表離線快取（⑤）

```ts
// 旅程列表快照（localStorage）：每次線上載入成功就覆寫，離線時唯讀呈現。

import type { TripWithMembers } from '../types'

const KEY = 'cotravel.tripsCache'

export interface TripsCache {
  trips: TripWithMembers[]
  cachedAt: string
}

export function saveTripsCache(trips: TripWithMembers[]): void {
  try {
    localStorage.setItem(KEY, JSON.stringify({ trips, cachedAt: new Date().toISOString() }))

    // 非致命
  }
}

export function loadTripsCache(): TripsCache | null {
  try {
    const raw = localStorage.getItem(KEY)

  } catch {
    return null
  }
}
```
