## 實作建議順序原則

1. 先打通「能登入 → 看到一趟旅程 → 在地圖加一個定點 → 對方看得到」這條主幹，再加旁支。
2. 交通、文件、行李彼此獨立，可平行或依興趣調序。
3. 即時同步留到核心 CRUD 穩定後再加，避免早期一直處理同步衝突。
