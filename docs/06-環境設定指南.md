# 06 · 環境設定指南

跨天接續或新環境上手時，照這份走。指令以 Windows PowerShell 為主。

## 1. Supabase 專案

1. 到 https://supabase.com 註冊（免費、不綁卡），建立新 Project。
2. 記下 **Project URL** 與 **anon public key**（Project Settings → API）。
3. 之後在 SQL Editor 貼上 `docs/03-資料模型.md` 的建表 SQL 與 RLS policy 執行。
4. 建立 Storage bucket（例如 `documents`），設為非公開，存取走 RLS。
5. **閒置提醒**：免費專案 7 天無活動會暫停，回來時在 Dashboard 手動 Restore 即可（資料不會掉）。

### 認證：Google 登入（OAuth）

登入以 **Google 為主**、Email+密碼為後備（決策見 `docs/05` 與專案 memory）。設定分三邊：

**A. Google Cloud Console**（https://console.cloud.google.com）

1. 沿用 Maps 那個專案即可。APIs & Services → **OAuth consent screen**：User Type 選 External，填 App 名稱與 support email；測試階段把要登入的兩個 Google 帳號加進 **Test users**（免送審）。
2. **Credentials → Create Credentials → OAuth client ID**，Application type 選 **Web application**。
3. **Authorized redirect URIs** 填 Supabase 的 callback——到 Supabase 的 Google provider 頁照抄它顯示的網址（格式 `https://<你的-ref>.supabase.co/auth/v1/callback`）。
4. 取得 **Client ID** 與 **Client Secret**（Secret 等下貼進 Supabase，**絕不寫進這份文件或程式碼**）。

**B. Supabase Dashboard**

1. Authentication → **Providers → Google**：貼上 Client ID / Client Secret，Enable。
2. Authentication → **URL Configuration → Redirect URLs**：開發與正式網址都要加進白名單，否則回跳被擋：
   ```
   http://localhost:5173/
   https://hfuhsu108.github.io/CoTravel/
   ```
3. （後備）Providers → **Email** 啟用；測試期可在 Authentication → Settings 關掉 **Confirm email**，免收信即可註冊登入。

**C. 前端 client（已設好，換機不用改）**

- `src/lib/supabase.ts` 必須設 `flowType: 'pkce'`。本專案用 **HashRouter**，OAuth 預設的 implicit flow 會把 token 放在網址 `#hash`、一回跳就被路由的 `Navigate` 蓋掉，登入永遠失敗；PKCE 把授權碼改放 `?code=` query 才能與 `#` 共存。

> ⚠️ 上正式網域或改 repo 名時：回 **B-2** 把新網域加進 Redirect URLs，並同步 Google Cloud 的 redirect URI 與 Vite `base`。完整坑況見專案 memory `gotcha-hashrouter-oauth-pkce`。

## 2. Google Maps 金鑰（要綁卡，務必設防護）

1. 到 https://console.cloud.google.com 建立專案，啟用帳單帳戶（需信用卡）。
2. **啟用 API**：Maps JavaScript API、Places API、Directions API。
3. 建立 API 金鑰。
4. **務必做兩件防護**（兩人自用避免被盜刷）：
   - **API 金鑰限制**：限定 HTTP referrer、限定只能用上面三個 API。referrer 填：
     ```
     http://localhost:5173/*
     https://hfuhsu108.github.io/*
     ```
     （開發初期可先只填 localhost 那條；referrer 可隨時回來修改，部署後再加 github.io。）
   - **預算上限 + 警報**：Billing → Budgets & alerts 設一個低額預算（如 NT$300）與 email 警報。
5. 記下金鑰填入 `.env`。

> 提醒：前端用的 Maps 金鑰本來就會出現在瀏覽器，**安全靠的是 referrer + API 限制**，不是把金鑰藏起來。

## 3. 環境變數

專案根目錄建立 `.env`（**不要 commit**），可先放一份 `.env.example` 當範本：

```
# .env.example
VITE_SUPABASE_URL=your-project-url
VITE_SUPABASE_ANON_KEY=your-anon-key
VITE_GOOGLE_MAPS_API_KEY=your-maps-key
```

## 4. 安裝與啟動

```powershell
npm install
# 若遇 UnauthorizedAccess / ExecutionPolicy 錯誤，改用：
#   cmd /c npm install
# （不要改系統 ExecutionPolicy）

npm run dev
```

dev server 起來後，用瀏覽器或 Claude Preview 開 `http://localhost:5173`（Vite 預設）驗證。

## 5. 部署到 GitHub Pages（之後）

帳號 `hfuhsu108`，網址為 `https://hfuhsu108.github.io/<repo>/`。

1. 建立 GitHub repo（例如 `cotravel`），推上去。
2. **Vite 設 `base: '/<repo>/'`**（如 `/cotravel/`），否則資源路徑會錯、重整會 404。
3. 前端路由用 HashRouter，或 BrowserRouter 設對應 basename。
4. 用 GitHub Actions 自動 build 並發布到 Pages（Settings → Pages → Source 選 GitHub Actions）。
5. **環境變數**：GitHub Pages 是純靜態，build 時 Vite 會把 `VITE_*` 變數編進前端。把三個變數存進 repo 的 **Actions secrets**，在 workflow build 步驟注入。
   - 注意：`anon key` 與 Maps 金鑰本來就會出現在前端，安全靠 RLS 與 referrer 限制，不靠隱藏。
6. 部署後把 `https://hfuhsu108.github.io/*` 加進 Google Maps 金鑰的 referrer 白名單。

## 常見問題

| 症狀 | 處理 |
|---|---|
| `npm install` 報 ExecutionPolicy / 指令碼已停用 | 改用 `cmd /c npm install` 或 Git Bash |
| 地圖空白、Console 報 `RefererNotAllowed` | 把目前網域/`localhost` 加進金鑰 referrer 白名單 |
| 部署後資源 404 或白畫面 | Vite `base` 沒設成 `/<repo>/`，或路由 basename 不對 |
| Supabase 連不上、查詢空 | 確認專案沒被閒置暫停；確認 RLS policy 有讓成員讀取 |
| 文件上傳失敗 | 確認 Storage bucket 存在、RLS/policy 正確、走的是 `src/lib/storage/` 抽象層 |
